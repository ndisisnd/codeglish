# Architecture mode

Runs when Step 1 of SKILL.md detects `--architecture`. Replaces the main Steps 2–6 for this invocation only. Emit `Architecture X/5 — <title>` at the start of each step, unconditionally.

An optional path argument may follow the flag (e.g. `--architecture src/`). When present, scope the scan to that directory; when absent, scan from the working directory root.

---

**Architecture 1/5 — Scan and classify files**

Run via Bash:
```bash
git ls-files 2>/dev/null || find . -type f -not -path './.git/*'
```

If a scope path was given, filter the output to paths that start with that prefix.

Apply the extension mapping and exclusion rules from `refs/codeglish-map.md` to every path in the result:
- Map each file to its canonical language using the extension table.
- Drop any file whose extension matches the exclusion list (images, fonts, media, archives, lock files, minified artifacts, source maps, `.env` files).
- Do not apply the noise threshold here — every individual file is included regardless of how many other files share its language.
- For unmapped extensions, treat them as language `Unknown` and include them in the output; do not ask the user to name them during this mode.

Sort the surviving file list by directory path, then by filename within each directory.

Output: `file_list` — an ordered list of objects, each with `{ path, language }`.
Also emit: `"Found N files across M languages."` inline before continuing.

---

**Architecture 2/5 — Read and describe each file**

For each file in `file_list`, read its contents using the Read tool.

- For files with 200 lines or fewer, read the entire file.
- For files with more than 200 lines, read only the first 200 lines (pass `limit: 200` to Read).

From the contents, write a single plain-English sentence that describes what the file does, using the "does X, so that Y" framing where both halves are natural:

- Good: `"Defines the database connection pool, so that every route handler shares a single managed connection."`
- Good: `"Entry point — starts the HTTP server on the configured port and mounts all route groups."`
- Bad: `"This file contains code."` — never write descriptions at this level of abstraction.

If the file is empty or unreadable, write: `"Empty file."`

Aim for one sentence per file. Two is acceptable only when a single sentence would omit something critical (e.g. a file that both exports a type and registers a side effect).

Output: `file_descriptions` — a map of `{ path → description }`.

---

**Architecture 3/5 — Build the architecture document**

Construct a markdown document with the following structure:

```
# Codebase Architecture

> Generated by Codeglish on <date>. Describes every code file found in <scope path or "the project root">.

## Summary

| Stat | Value |
|------|-------|
| Total files | N |
| Languages | L1 (n files), L2 (n files), … |
| Scope | <scope path or "entire project"> |

---

## File Index

<one section per top-level directory, ordered alphabetically>

### <directory name or "Root" for top-level files>

| File | Language | What it does |
|------|----------|-------------|
| `path/to/file.ext` | Language | Description sentence. |
| … | … | … |

<repeat for each directory>
```

Rules for the File Index:
- Use the relative path as it appears in the file list.
- Group files by their immediate parent directory. Files in the project root (no subdirectory) go under a `### Root` heading.
- Nested directories each get their own section, even if they contain only one file.
- Within a directory section, sort rows by filename.
- Paths in the `File` column are wrapped in backticks so they render as inline code.
- Descriptions come directly from `file_descriptions`. Do not rephrase or shorten them here.

Output: `architecture_doc` (full markdown text, not yet written to disk).

---

**Architecture 4/5 — Offer write options**

Show the architecture document inline.

Then ask via AskUserQuestion:

> "Where would you like to save the architecture file?"
> - **Save as `codeglish-architecture.md`** — write to the project root (default)
> - **Show only** — keep it in the chat, no file written

If the user chooses **Save as `codeglish-architecture.md`**: write the document to `codeglish-architecture.md` at the project root using the Write tool.
If the user chooses **Show only** (or dismisses): no file write.

Output: chosen option; file path if written.

---

**Architecture 5/5 — Confirm**

Emit a one-line confirmation:

- If file was written: `"Architecture saved to codeglish-architecture.md — N files documented."`
- If show only: `"Architecture shown inline — N files documented."`

No XP is awarded in architecture mode; this step has no `refs/codeglish-exp.json` write.

Output: confirmation line shown inline.
